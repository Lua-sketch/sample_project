Bits e Representação Binária¶

Um bit uma unidade de informao clssica. Seu nome vem de dgito binrio e cada bit pode adotar um dentre dois valores: 0 ou 1. Todas as operaes que ocorrem em um computador so baseadas em manipulaes de bits.

Para ns, basta pensar em um bit como a nossa menor unidade de informao, um objeto matemtico que representaremos com vetores. Fisicamente, um bit implementado com eletricidade, sendo que ele vale 0 quando no deixa a eletricidade passar e 1 quando o faz.

Bom, dissemos que qualquer operao em um computador resultado da manipulao de bits. Assim, a realizao de uma soma ou de qualquer outra operao aritmtica deve poder ser feita com bits. Mas como?

Vamos agora relembrar como funciona nosso sistema nmerico comum, de base dez. O nmero 1875 pode ser expresso como

    1*1000 + 8*100 + 7*10 + 5*1 = 1*10 + 8*10 + 7*10 + 5*10

Assim, se damos posies aos algarismos que compem nossos nmeros inteiros a partir do zero, comeando da direita, vemos que cada um deles representa o coeficiente pelo qual multiplicamos 10, onde pos sua posio.

Se temos nmeros decimais, introduzimos posies negativas:

  1875.43 = 1*10 + 8*10 + 7*10 + 5*10 + 4*10 + 3*10

Para realizar o mesmo tipo de operao que representamos com nosso sistema de base 10 usando bits, precisamos introduzir uma lgica similar sobre como ler os algarismos em uma sequncia de bits. Ento, criamos a base binria ou base 2, em que as posies agora correspondem aos coeficientes pelos quais multiplicamos 2.

Vejamos alguns exemplos:

     100 = 1*2 + 0*2 + 0*2 = 4
     1001 = 1*2 + 0*2 + 0*2 + 1*2 = 9

Por fim, antes de aprendermos aritmtica em base binria, vejamos como podemos converter um nmero da base 10 para a base 2.

Para isso, vamos utilizar o exemplo do nmero 25. Antes, dividimos o nmero por 2:

      25/2 = 12, com resto 1

Ento, escrevemos o resto dessa primeira diviso na posio do coeficiente de 2:

          1

Agora, dividimos o resultado da diviso por 2:

    12/2 = 6, com resto 0

Desta vez, escrevemos o resto da diviso na posio do coeficiente de 2:

       01

Mais uma vez, dividimos o resultado por 2:

  6/2 = 3, com resto 0

E o resto vai na posio do coeficiente de 2:

      001

O processo se repete:

3/2 = 1, com resto 1

1001

1/2 = 0 com resto 1

11001 -> expresso final, pois chegamos no resultado zero!

Se quiser, pode checar que a expresso est correta. De qualquer forma, este algoritmo sempre funciona, e apesar de ser possvel chegar a um resultado sem ele, apenas por meio de tentativa e erro, ele garante eficincia e constncia, alm de ajudar quando os nmeros so muito grandes.

Agora, vamos reaprender aritmtica! Em nossa base decimal, dgitos eram "carregados" na adio quando eram maiores ou iguais a 10. Agora, estamos em base binria, lidando com bits, ento no podemos ter dgitos maiores que 1! Os carregamos para a frente se so maiores ou iguais a 2. Vejamos alguns exemplos:

1 + 1 = 10
11 + 1 = 100
111 + 111 = 1110

A subtrao segue a mesma lgica, e no vamos precisar aprender sobre multiplicao ou diviso binria. Basta saber que

0 X 0 = 0

0 X 1 = 0

1 X 0 = 0

1 X 1 = 1

0/0 = 0

0/1 = 0

1/1 = 1

Lógica Booleana!¶

Para terminar nossa base terica, vamos falar sobre lgica booleana!

A lgica booleana nada mais que o estudo de funes booleanas, que mapeiam argumentos (em forma de bits) a dois possveis resultados (0 ou 1). No corao dessas funes est a ideia de que tudo verdadeiro (1) ou falso (0).

Existem trs funes/operadores booleanos principais: "AND", "OR" e "NOT". A operao "AND" s resulta no valor "verdadeiro" ou 1 quando ambos os argumentos so verdadeiros. Por exemplo, se digo que "1 + 1 = 2 AND 3 + 4 = 7", minha afirmao verdadeira e retorna o valor 1. No entanto, se digo que "1 + 1 = 2 AND 3 + 4 = 8", apesar da primeira parte da afirmao ser verdadeira, a afirmao como um todo falsa.

J a operao "OR" retorna 1 quando pelo menos um dos argumentos verdadeiro. Assim, tanto "1 + 1 = 2 OR 3 + 4 = 7" quanto "1 + 1 = 2 OR 3 + 4 = 8" so verdadeiras.

Por fim, a operao "NOT" apenas nega o valor do argumento que recebe. Assim, se recebe algo como "1 + 1 = 2 e 3 + 4 = 8", que falso, retorna verdadeiro/1.

Para formalizar um pouco esse tipo de operao, podemos considerar como argumentos apenas valores de falso (0) ou verdadeiro (1). Dessa forma, no lugar de falar que a operao "AND" retorna 1 quando ambas as partes de uma afirmao so verdadeiras, e fornecer vrios exemplos, podemos simplesmente dizer que "1 AND 1 = 1" (verdadeiro AND verdadeiro = verdadeiro).

O mesmo se aplica a todas as outras operaes. Veja que o que construmos so funes de bits para bits! Outro nome para funes/operaes booleanas portas lgicas. Existem vrias outras dessas "portas", que podemos resumir segundo a tabela abaixo:
